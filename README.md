# 🐖 智慧养殖溯源管理系统

## 📖 项目介绍

智慧养殖溯源管理系统是一个基于Spring Boot的企业级养殖管理平台，实现了从动物养殖、批次管理、疾病防控到检疫登记的全流程数字化管理。系统通过现代化技术手段，为养殖企业提供高效、精准、可追溯的管理解决方案。

## 🚀 项目功能

### 🔐 用户认证模块

- JWT Token身份认证
- 用户登录/注销
- 权限拦截控制

### 📊 数据大屏模块

- **资源统计**：栏舍、栏圈、动物、冷库、员工数量实时统计
- **销售分析**：本年度12个月动物销量趋势分析
- **疾病监控**：各类病症患病动物数量统计
- **体重分布**：按体重区间(30以下/30-50/50以上)统计动物数量

### 🏠 栏舍栏圈管理

- **栏舍管理**：栏舍信息CRUD、批量删除、关联栏圈查询
- **栏圈管理**：栏圈状态控制(启用/禁用)、容量管理、空满状态监控
- **智能分配**：自动推荐可用栏圈，防止过度拥挤

### 🐷 动物管理模块

- **动物档案**：动物基本信息、健康状态、接种情况管理
- **批次关联**：动物与批次信息关联，自动同步检疫状态
- **动态迁移**：动物栏圈变更自动更新栏圈容量状态

### 🏥 健康管理模块

- **病症库**：标准病症信息维护(病因、症状、预防措施)
- **治疗记录**：病症治疗过程记录，支持未治疗/治疗中/已治疗状态
- **健康联动**：治疗状态自动更新动物健康状态

### 🔬 检疫管理模块

- **批次检疫**：按批次进行检疫登记，支持图片上传
- **检疫机构**：多检疫机构记录管理
- **结果追溯**：检疫合格/不合格状态全程记录

### 📦 批次管理模块

- **批次跟踪**：动物批次全生命周期管理
- **检疫状态**：未检疫/已检疫状态自动同步
- **数据关联**：批次与动物、检疫记录的完整关联

## 🛠 技术栈

### 后端技术

- **核心框架**：Spring Boot 2.7.6
- **数据层**：MyBatis + MyBatis PageHelper分页
- **数据库**：MySQL + Druid连接池
- **安全认证**：JWT Token认证
- **API文档**：Knife4j (Swagger)

### 开发工具

- **构建工具**：Maven
- **Java版本**：JDK 1.8
- **代码简化**：Lombok
- **文件存储**：阿里云OSS

### 架构特色

- **分层架构**：Controller-Service-Dao标准分层
- **统一响应**：Result结果对象统一封装
- **全局异常**：ServiceException + GlobalExceptionHandler
- **配置集中**：YAML配置文件管理

## 🚀 快速开始

### 环境要求

- JDK 1.8+
- MySQL 5.7+
- Maven 3.6+

### 启动步骤

1. 克隆项目到本地
2. 创建数据库`product_source`并导入SQL脚本
3. 修改`application.yml`中的数据库配置
4. 运行`mvn install`安装依赖
5. 启动`ProductSourceApplication`主类
6. 访问`http://localhost:9999/doc.html`查看接口文档

### 接口测试

1. 调用`/login`接口获取Token
2. 在请求头中添加`token: {jwt-token}`
3. 通过Knife4j界面测试各功能接口

## 📁 项目结构

text

```
src/main/java/com/briup/product_source/
├── config/          # 配置类(Web配置、Swagger等)
├── controller/      # 控制层(RESTful接口)
├── service/         # 业务逻辑层(接口+实现)
├── dao/            # 数据访问层(Mapper接口)
├── pojo/           # 实体类(基础实体+扩展实体)
├── exception/      # 异常处理(自定义异常+全局处理)
├── interceptor/    # 拦截器(JWT认证拦截)
├── result/         # 统一返回结果
└── util/           # 工具类(JWT工具等)
```
